## 한 줄

동그라미재단에서 운영한 **프롬프트 엔지니어링 교육**을 이수했습니다. LLM의 행동을 의도대로 만들기 위해 프롬프트 단계에서 어떤 설계가 필요한지를 다룬 과정이었습니다.

## 무엇을 배웠나

- LLM의 행동 변화를 일으키는 핵심 요소 (시스템 지시, 사용자 입력, 컨텍스트, 도구)
- 의도된 행동을 안정적으로 끌어내는 프롬프트 구조 설계
- 실제 시나리오에 맞춰 프롬프트를 다듬는 반복 과정

## OH! GYM과 연결되는 점

프롬프트 엔지니어링은 **"에이전트가 어떻게 행동하길 원하는지를 명세하는 일"**입니다. 에이전트가 내가 원하는 행동을 하도록 만들기 위한 프롬프트 구조를 얻기 위해 여러 입력을 수천번 또는 그 이상 반복하여 LLM의 출력을 검증합니다. 강화학습의 보상함수 설계와 본질적으로 같은 문제를 풀고 있다고 생각됩니다.

이 경험은 휴머노이드 RL에서 reward shaping을 고민할 때, **"행동을 정의하는 일은 어렵고 반복적이다"**라는 감각을 가지게 해줍니다.

## One line

Completed the **prompt engineering program** run by the Donggurami Foundation — covering how to design prompts so that LLMs behave the way you intend.

## What I learned

- Key levers that shape LLM behavior (system instructions, user input, context, tools)
- Prompt structures that reliably elicit the intended behavior
- The iterative process of tuning prompts against real scenarios

## Connection to OH! GYM

Prompt engineering is **"specifying how you want an agent to behave."** To find a prompt structure that makes an agent do what I want, I iterate through hundreds or thousands of inputs and verify the LLM's outputs. I think this is essentially the same problem as designing a reward function in reinforcement learning.

This experience gives me the instinct, going into reward shaping for humanoid RL, that **defining behavior is hard and iterative**.
